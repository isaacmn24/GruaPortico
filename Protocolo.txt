Protocolo UART puede enviar un byte.
Se toma Raspberry Pi (RBP) como master (cerebro) y STM32 como slave.
Asi, las comunicaciones necesarias son las siguientes:
1. Enviar "pasos" necesarios a cada motor (son dos).
2. Enviar señal de activación y desactivación del electroimán.
3. Solicitar sensor de color.
4. Solicitar sensor de distancia.

El STM32 estará constantemente esperando una señal del RBP, por lo que el que comienza la comunicación es el master. Entonces, se propone el siguiente protocolo:
1. Envía instrucción.
	0x01	->	Sensor color
	0x02	->	Sensor distancia
	0x03	->	Electroimán ON
	0x04	->	Electroimán OFF
	0x05	->	Motor xx
	0x06	->	Motor yy

2. Para los sensores, master no enviará nada, si no que esperará una señal del slave.
	a. Color: Recibirá tres señales seguidas de dos bytes, correspondientes a las señales R, G y B.

3. Para los actuadores, el master puede o no enviar una señal después de la instrucción.
	a. Electroimán: No envía ninguna señal extra, solo espera un "recibido".
	b. Motores: Si manda una segunda señal de dos bytes, correspondiente a los pasos que debe tomar el motor. 
	Además, espera una señal de "recibido".
